Rule-Based (No AI needed):
┌─────────────────────────────────────────────────────────────┐
│ ✓ Characters with titles (King, Queen, Lord, etc.)         │
│ ✓ Proper nouns with character context                      │
│ ✓ Timeline events (wars, ages, battles)                    │
│ ✓ Dates (years, ages, temporal markers)                    │
│ ✓ Topics/themes (Magic, War, Politics, etc.)               │
│ ✓ Locations (kingdoms, cities, mountains)                  │
│ ✓ Matches against existing vault entries                   │
│ ✓ Gender inference from titles                             │

🔌 INTEGRATION WITH YOUR APP:

  • Add a "Scan Document" button to your app
  • Show suggestions in a modal
  • Let users approve/reject suggestions
  • Auto-add entities to vault

Quick integration steps:
  1. Add IPC handler to main.js (provided)
  2. Add UI button to index.html
  3. Add scanner class to renderer
  4. Done! Users can now auto-scan their documents


📈 PERFORMANCE:

Rule-Based:
  • Instant (< 1 second for most documents)
  • No API calls or internet required
  • Works offline

AI-Enhanced:
  • 2-10 seconds (depends on service/model)
  • Requires API key or local model
  • Better accuracy


💡 TIPS:

• Start with rule-based to see what it can do
• Documents with 500+ words work best
• Proper nouns and titles improve detection
• Review suggestions before adding to vault
• Adjust confidence thresholds in code if needed
